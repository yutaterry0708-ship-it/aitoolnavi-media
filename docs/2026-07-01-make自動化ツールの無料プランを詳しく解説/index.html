「無料で自動化できるって本当？」——Makeの無料プランを使い倒した結論

「Zapierは高すぎる」「でも自動化ツールは試してみたい」——そう思ってMakeの無料プランを触り始めた方は多いはずです。私自身、複数のノーコード自動化ツールを仕事で比較検証してきた中で、Makeの無料枠は「ちゃんと実用できる珍しい無料プラン」だと感じています。

ただし、制限を理解しないまま使い始めると「思ったより動かない」「いつの間にか上限に達した」という落とし穴にはまります。

この記事では、Makeの無料プランで実際に何ができて何ができないのか、競合ツールとの違い、有料プランへの移行を検討すべきタイミングまで、実体験を交えて正直にお伝えします。

Makeとは何か——まず30秒でおさらい

Make（旧称：Integromat、2022年にリブランド）は、チェコ発のノーコード自動化プラットフォームです。複数のWebサービスを「シナリオ」と呼ばれるフロー図で繋ぎ、繰り返し作業を自動化できます。

視覚的なキャンバス上でモジュールをドラッグ＆ドロップして繋ぐUIが特徴で、条件分岐・ループ・エラーハンドリングをGUI上で組める点がZapierとの最大の差別化ポイントです。実際に使ってみて、「ちょっと複雑なロジック」を組みたい場面でMakeのほうが圧倒的に直感的だと感じました。

Makeの無料プランで使える機能【2025年最新】

無料プランの基本スペック

Makeの無料プランは、クレジットカード登録不要でスタートできます。主な仕様は以下のとおりです。

項目 無料プランの内容
月間オペレーション数 1,000オペレーション/月
アクティブシナリオ数 2シナリオまで
最小実行間隔 15分ごと
データ転送量 100MB/月
アプリ連携数 1,000以上（制限なし）
チームメンバー 1名（本人のみ）
シナリオの複雑さ 制限なし（ステップ数・分岐数に上限なし）

「オペレーション」とは、シナリオ内で各モジュールが1回処理を行うことを指します。たとえば「Gmailを受信してGoogleスプレッドシートに書き込む」という2ステップのシナリオが1回走ると、2オペレーション消費します。

無料でも使えるアプリ連携

無料プランでも1,000以上のアプリ連携（コネクター）が使えます。Google Workspace、Slack、Notion、Airtable、OpenAI、Webhookなど、実務でよく使うサービスはほぼカバーされています。「プレミアムアプリ」という有料限定コネクターが一部ありますが、日常業務で困るケースは少ないというのが私の実感です。

実際に無料プランで動かせる自動化の例

私の運用では、以下のようなシナリオを無料プランで問題なく稼働させていました。

  • Googleフォームの回答をNotionデータベースに自動登録（2モジュール、15分ごと実行）
  • RSSフィードの新着記事をSlackに通知（3モジュール）
  • GmailでラベルをつけたメールをGoogleスプレッドシートに記録（3モジュール）

月1,000オペレーションという制限は、「1日に数回動かす軽い自動化を2本」という使い方なら十分に収まります。逆に、Eコマースの注文処理やリード管理など、1日に何十件もデータが流れる業務には明らかに足りません。

  • Zapierの無料プランは「100タスク/月」かつ「2ステップまで」という制限が厳しく、実務では使い物にならないことが多いです。私が実際に試したときも、3ステップのシナリオを組もうとした時点で有料プランへの誘導が表示されました。

Makeの無料プランの正直なメリット・デメリット

✅ メリット

1. 複雑なシナリオを無料で組める
条件分岐（Router）、繰り返し（Iterator/Aggregator）、エラーハンドリングが無料プランでも制限なく使えます。ZapierやIFTTTでは有料プランでしか触れない機能です。

2. クレジットカード不要で始められる
メールアドレスだけで登録でき、無料期間終了後に自動課金される心配がありません。試しやすさは業界トップクラスです。

3. 1,000以上のアプリ連携が無料から使える
OpenAIやNotion、Webhookなど、最近注目のツールとの連携も無料プランで対応しています。

4. シナリオのデバッグ機能が使いやすい
実行ログが視覚的に確認でき、どのモジュールでエラーが起きたかが一目でわかります。これは初心者にとって非常に助かります。

❌ デメリット（正直に書きます）

1. 月1,000オペレーションは思ったより早く消える
1シナリオあたり5モジュール使っていて、1日20回実行するだけで月に3,000オペレーション消費します。業務量が増えると、あっという間に上限に達します。

2. 最小実行間隔が15分
リアルタイム性が求められる用途（注文通知、チャットBot応答など）には向きません。Webhookトリガーを使えばリアルタイム実行も可能ですが、設定の難易度が上がります。

3. アクティブシナリオが2つまで
複数の業務を自動化しようとすると、すぐに上限に達します。私の運用では「もう1本組みたい」と思った時点で有料プランへの移行を検討しました。

4. チーム利用は有料プランから
複数人でシナリオを管理・共有したい場合は、Coreプラン以上が必要です。フリーランスの個人利用なら問題ありませんが、チームでの導入を検討している場合は注意してください。

5. UIの学習コストがやや高い
Zapierに比べてMakeのUIは独特で、初見では戸惑うユーザーも多いです。「モジュール」「バンドル」「オペレーション」といった独自の概念を理解するまでに時間がかかります。

有料プランへの移行を検討すべきタイミング

以下のいずれかに当てはまる場合は、有料プランへの移行を真剣に検討する価値があります。

  • 月1,000オペレーションを毎月使い切っている
  • 3本以上のシナリオを常時稼働させたい
  • チームメンバーとシナリオを共有したい
  • 5分以下の実行間隔が必要な業務がある

Makeの有料プランは「Core」「Pro」「Teams」「Enterprise」の4段階。Coreプランは月額約9ドル（年払い）から始まり、オペレーション数が10,000に増え、アクティブシナリオも無制限になります。コスパは競合と比べて高めです。

よくある質問（FAQ）

Q1. Makeの無料プランは期限切れになりますか？

A. 無料プランに期限はありません。ただし、連続して3ヶ月以上ログインしないとアカウントが非アクティブ化される場合があります。定期的にログインするか、シナリオを稼働させておくことで維持できます。

Q2. 無料プランでWebhookは使えますか？

A. 使えます。Webhookを受信トリガーとして設定すれば、15分間隔の制限を回避してリアルタイムに近い実行が可能です。ただし、Webhookの設定には多少の技術的な理解が必要です。無料プランでも試せるので、まずは公式ドキュメントを参照しながら挑戦してみてください。

Q3. ZapierからMakeに乗り換えるメリットはありますか？

A. 「複雑な条件分岐やループを組みたい」「コストを抑えたい」という方には乗り換えのメリットがあります。一方、「シンプルな2ステップ自動化をたくさん作りたい」「Zapierのアプリ連携数に依存している」場合はZapierのほうが合っている場面もあります。実際に私が両方を使った印象では、ロジックの複雑さに比例してMakeの優位性が高まると感じています。

まとめ——Makeの無料プランは「本物の入口」

Makeの無料プランは、「お試し用の制限版」ではなく、軽い自動化なら実務で使い切れる設計になっています。特に「複雑なロジックを視覚的に組める」という点は、無料プランでも変わりません。

一方で、月1,000オペレーション・2シナリオという制限は現実的な壁です。業務が広がるにつれて有料プランへの移行は避けられませんが、まず無料で試して、自分の用途に合うか確かめてから課金するという使い方が最もリスクが低いでしょう。

自動化ツールを初めて触る方にも、Zapierから乗り換えを検討している方にも、Makeの無料プランは「試す価値がある」と自信を持っておすすめできます。
