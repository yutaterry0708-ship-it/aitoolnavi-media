「補完が出ない」「どこで設定するの？」――Copilotで詰まる人が多い理由

GitHub Copilotを契約したはいいものの、「VS Codeで全然サジェストが出ない」「Chat機能はどこにある？」「拡張機能が2つあって混乱する」――こうした声はコミュニティやSNSで今でも頻繁に見かけます。

公式ドキュメントは英語が多く、UIのアップデートも早いため、半年前の記事を見ながら設定しようとすると画面が全然違う、ということも珍しくありません。

この記事では、実際にGitHub CopilotをVS Codeで日常的に使い込んでいる筆者が、2025年現在の最新UIに合わせた導入手順・推奨設定・正直なデメリットまでをまとめて解説します。 読み終わる頃には「とりあえず動く状態」から「快適に使える状態」まで到達できるはずです。

GitHub Copilotとは？VS Codeとの相性が抜群な理由

GitHub Copilotは、OpenAIのモデルをベースにGitHubが開発したAIコーディングアシスタントです。コードの自動補完だけでなく、チャット形式での質問・コードレビュー・テスト生成など、開発ワークフロー全体をサポートします。

VS Codeとの相性が特によい理由は主に3点です。

  • 公式拡張機能が充実：GitHubが直接メンテナンスしているため、機能追加が最も早い
  • インラインChat・Edits機能：ファイルを開いたまま自然言語で修正依頼できる
  • エージェントモード（2025年〜）：複数ファイルにまたがる変更を自律的に実行できる

実際に使ってみて感じるのは、JetBrainsやNeovimプラグインと比べて「UIの洗練度」が一段上だという点です。特にInline Chatのショートカット（Ctrl+I / Cmd+I）はコードを書く手を止めずに質問できるため、一度慣れると手放せなくなります。

導入手順：VS CodeにGitHub Copilotを設定する

ステップ1：GitHubアカウントとプランの準備

  1. github.com にアクセスし、アカウントを作成またはログイン
  2. 右上のアバター → Settings → Copilot から希望プランを有効化
  3. Freeプランは自動で有効、有料プランはクレジットカード登録が必要

ステップ2：VS Code拡張機能のインストール

VS Codeの拡張機能マーケットプレイスで 「GitHub Copilot」 と検索します。

注意点：検索結果に似た名前の拡張機能が複数表示されます。インストールすべきは以下の2つです。

拡張機能名 役割
GitHub Copilot インライン補完（コード自動補完）
GitHub Copilot Chat チャット・Inline Chat・Edits機能

2025年現在、「GitHub Copilot Chat」は「GitHub Copilot」をインストールすると自動的に同梱されるケースが多いですが、古いVS Codeバージョンでは別途インストールが必要なこともあります。バージョンは1.85以上を推奨します。

ステップ3：GitHubアカウントでサインイン

拡張機能インストール後、VS Code左下のアカウントアイコン → 「GitHub Copilotでサインイン」 をクリック。ブラウザが開いてOAuth認証が完了すれば、ステータスバーにCopilotのアイコンが表示されます。

詰まりポイント：サインイン後もアイコンが「×」のままの場合、プロキシ環境やVPN接続が原因のことがあります。settings.jsonに"http.proxy"を設定するか、VPNを一時的にオフにして確認しましょう。

ステップ4：動作確認

任意のファイル（例：test.py）を開き、以下を入力してみてください。

def fibonacci(n):

数秒でグレーのサジェストテキストが表示されれば成功です。Tabキーで確定、Escで却下できます。

快適に使うための推奨設定（settings.json）

インストール直後のデフォルト設定でも動きますが、私の運用では以下の設定を加えることで格段に使いやすくなりました。

{
  // インライン補完の有効化（デフォルトtrue、念のため明示）
  "github.copilot.enable": {
    "*": true,
    "markdown": true,
    "plaintext": false  // テキスト編集中の誤爆を防ぐ
  },

  // 補完候補を複数表示（Copilot Labsが不要になった機能）
  "editor.inlineSuggest.enabled": true,

  // Copilot Chatのデフォルトモデル指定（2025年時点）
  "github.copilot.chat.defaultModel": "gpt-4o",

  // エディタ内でChatパネルを開くショートカット確認用
  // Ctrl+Shift+I (Windows) / Cmd+Shift+I (Mac)

  // 保存時に自動フォーマット（Copilotと組み合わせると快適）
  "editor.formatOnSave": true
}

言語別にCopilotをオフにする方法

Markdownや設定ファイル編集中にCopilotが余計なサジェストを出してくる場合、"github.copilot.enable"の言語キーをfalseにするだけで制御できます。実際に使ってみると、YAMLファイルでの誤補完がかなりのストレスだったため、私の環境では"yaml": falseも追加しています。

主要機能の使い方

インライン補完（Ghosttext）

最も基本的な機能。コードを書くと自動でサジェストが表示されます。

操作 Windows/Linux Mac
サジェスト確定 Tab Tab
サジェスト却下 Esc Esc
次の候補 Alt+] Option+]
前の候補 Alt+[ Option+[
全候補をパネル表示 Ctrl+Enter Ctrl+Enter

Inline Chat（Ctrl+I / Cmd+I）

コードを選択した状態でショートカットを押すと、その場でCopilotに指示できます。

/fix このループのオフバイワンエラーを修正して
/explain このコードを日本語で説明して
/test ユニットテストを生成して

実際に使ってみて特に便利だったのは /explain です。他人のコードレビュー時に、難解な正規表現や複雑なアルゴリズムを即座に日本語で説明させると、読解時間が大幅に短縮できます。

Copilot Edits（複数ファイル編集）

VS Code 1.93以降で利用可能な機能。サイドパネルから「Copilot Edits」を開き、変更したいファイルを複数追加してから自然言語で指示します。

「このプロジェクトのエラーハンドリングをすべてtry-catchに統一して」といった横断的な変更を一度に提案させることができます。ただし、変更内容は必ず差分を確認してから適用することを強く推奨します。意図しないファイルまで変更されるケースがあるためです。

正直なメリット・デメリット

✅ メリット

  • 補完の精度が高い：特に定型的なコード（CRUD処理・テスト・設定ファイル）の生成は目を見張るレベル
  • コンテキスト理解が優秀：開いているファイルや関連ファイルを参照して補完するため、プロジェクト固有の命名規則に沿ったコードが出やすい
  • 学習コストが低い：インストールして使うだけ。特別な設定なしでも即日使える
  • 日本語でのChat対応：質問・指示を日本語で書いても精度の高い回答が返ってくる

❌ デメリット（正直に書きます）

  • 月$10のコストは人によって重い：フリーランスや副業開発者には費用対効果を感じやすいが、趣味プログラマーには割高に感じることも
  • 補完が「それっぽいが間違っている」コードを出すことがある：特に最新ライブラリのAPIや、ニッチなフレームワークでは古い・誤った補完が出る。鵜呑みにせず動作確認は必須
  • Freeプランの制限が体感でわかりやすい：月2,000回の補完は思ったより早く消費する。「今日は使えない」となるとストレスになる
  • プライバシー設定の確認が必要：デフォルトでは入力コードがモデル改善に使われる設定になっている場合があります。Business/Enterpriseプランでは無効化できますが、個人プランでは設定を確認・変更する必要があります（Settings → Copilot → Allow GitHub to use my code snippets）
  • ネット接続必須：オフライン環境では一切動作しません。ローカルLLMを使いたい場合はContinueなど別ツールを検討する必要があります

FAQ

Q1. GitHub Copilot無料プランとProプランは何が違いますか？

最大の違いは補完回数とChat回数の上限です。Freeプランは月2,000回の補完・50回のChatが上限で、超過すると月末まで使えなくなります。Proプランは$10/月で両方が無制限になり、エージェントモードも利用可能です。本格的に開発に使うならProプランへの移行を検討するのが現実的です。

Q2. 補完が全然出ないときのチェックポイントは？

以下の順番で確認してください。

  1. VS Codeのステータスバー右下にCopilotアイコンが表示されているか
  2. アイコンをクリックして「Enabled」になっているか（言語別にオフになっている可能性）
  3. GitHubアカウントのサインイン状態を確認（アカウントアイコンから再サインイン）
  4. プロキシ・VPN環境下でないか確認
  5. VS Codeを最新バージョンにアップデートする

それでも解決しない場合は、VS Codeのコマンドパレット（Ctrl+Shift+P）から「GitHub Copilot: Diagnose」を実行すると詳細なエラー情報が得られます。

Q3. 入力したコードがGitHubに送られてセキュリティリスクはありますか？

個人プラン（Free/Pro）のデフォルト設定では、コードスニペットがモデル改善のために送信される場合があります。機密プロジェクトで使用する場合は、GitHubのSettings → Copilot → 「Allow GitHub to use my code snippets for product improvements」をオフにすることを推奨します。Business/Enterpriseプランでは組織単位でこの設定を管理でき、コードがモデルトレーニングに使われないことが契約で保証されています。

まとめ

GitHub CopilotをVS Codeで使うための要点を整理します。

  • インストールは5分以内：拡張機能2つ＋GitHubサインインで完了
  • Freeプランでまず試す：月2,000回の補完で使用感を確認してからProへ移行
  • settings.jsonで言語別制御：Markdownや設定ファイルは誤補完が多いのでオフ推奨
  • Inline Chatは最強の武器：Ctrl+Iを使いこなすと生産性が大きく変わる
  • 補完を鵜呑みにしない：特に新しいAPIや外部ライブラリの補完は動作確認必須

実際に使ってみて、「コードを書く速度」よりも「調べる・考える時間」が減った実感の方が大きいです。エラーの原因調査・ドキュメント読み込み・テスト作成といった「本来の実装以外の作業」をCopilotに任せることで、本質的な設計や実装に集中しやすくなります。

まずはFreeプランで試してみて、物足りなくなったらProへ。それが最もリスクの少ない始め方です。
